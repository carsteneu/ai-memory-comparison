## Extraction Pipeline

| Feature | Value | Evidence |
|---------|-------|----------|
| Auto-extraction | true | Auto-generated memory: agent responses auto-written to KG via `write_back()` using `KnowledgeGraphIndex.from_documents()`. README principle: "Auto-generated Memory — After initializing memary, agent memory automatically updates as the agent interacts." |
| Content-aware preproc | false | No content-aware preprocessing |
| Deduplication | true | Entity Knowledge Store groups identical entities, incrementing count: `_convert_memory_to_knowledge_memory()` creates unique entity set, `_update_knowledge_memory()` increments existing counts |
| Quality refinement | false | No multi-pass quality refinement |
| Narrative generation | false | No narrative generation |
| Clustering | false | No entity/topic clustering. The README mentions "Categorize Entities: Group entities into categories" but this is aspirational — the source code only groups identical entities without semantic categorization. |
| Recurrence detection | true | Entity Knowledge Store tracks frequency (count) and recency (date). README: "Extract Themes: Look for recurring themes or topics within the interactions." Count-based relevance ranking detects frequently mentioned entities. |
| Persona extraction | true | System tracks user interests/preferences through Entity Knowledge Store (frequency + recency ranking). User persona template (`streamlit_app/data/user_persona_template.txt`) is manual but the system auto-builds an entity preference profile. Top entities injected into context window for personalized responses. |

## Additional Notes

### Synonym Expansion
Custom LLM-powered synonym expansion for KG queries (`synonym_expand/synonym.py`): uses LangChain + OpenAI to find synonyms for search keywords before KG retrieval.

### Context Window Management
Token-aware context eviction at 70% threshold (`EVICTION_RATE = 0.7`): when total tokens exceed `CONTEXT_LENGTH * 0.7`, older conversations are summarized into a 50-word compressed message.

### Default Tools
Built-in agent tools: search (KG + external), vision (LLaVA/GPT-4V), locate (geocoder + Google Maps), stocks (Alpha Vantage). Extensible via `add_tool()` / `remove_tool()`.

### Architecture Summary
```
User Query → ReAct Agent → KG Search (recursive/multi-hop) → External Fallback (Perplexity)
                ↓
         Response → Write-back to KG → Memory Stream → Entity Knowledge Store
                ↓
         Top Entities + Chat History → Context Window for Next Inference
```
